## Probable Causes

1. **Client-side disconnections / request timeouts** — The most likely root cause. Clients (load balancer health checks, upstream proxies, or end users) are dropping connections mid-request. The `CancellationToken` fires, cancels in-flight async operations, ASP.NET Core then disposes the `HttpContext`, and any code path that doesn't check `ct.IsCancellationRequested` before accessing the context throws `ObjectDisposedException`.

2. **Upstream proxy timeout mismatch** — If an API gateway or reverse proxy has a shorter timeout than the application, it closes the TCP connection while the app is still processing, producing the same pattern.

3. **Deployment / rolling restart at ~12:56 UTC** — Both exceptions appeared simultaneously at 12:56:19 UTC. A pod restart or slot swap would abruptly cancel in-flight requests en masse, creating an initial burst followed by a sustained tail if the underlying issue persists.
